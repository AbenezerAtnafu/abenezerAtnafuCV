Does it really matter on our styling to make the element responsive?

We all want to do many activities on the same window by giving them half of the screen or quarter of the screen. Then I am quite sure from one of those windows there will be one browser. Then you have may figure some flexible units of measurement, when some elements stack as they were in the initial stage.


These flexible and scalable units measurement is translated by our browser into the pixel values, that's why we are seeing different sizes with the ratio to our windows size. %, em and rem are the tools that are used to make flexible, scalabel units. They give the right to the browser in order to control the pixel values. They give us the ability to scale elments up and down. Understanding these key points are the basic thing to use each unit:

What is the crucial difference between them?

The difference between %, em and rem units is used to tell the browser how to determine the pixel value to translate into.

Translation of rem units to pixel is determined by the font size of the html element. Which means it is influenced by the browser font size setting.

Translation of em units to pixel values is determined by the size of the element they are used in. The size is influenced by the inheritance from the parent element.

The usage of the % is the same with em.


When to use them?

When to use rem?

Since the rem units offer us a way to have user font size settings, the primary purpose of using rem units should be to ensure that whatever default font size a user has their browser set to, the layout adjust to accommodate the size of text within it. This will help for persons that has a problem of strained eyesight. Don't forget that your site best arrangement will be affected by the user's browser setting.


When to use em?

The key concept of em is that they allow sizing values to be deternmined by the font size of another element (it's parent element). For this reason, the em units goes with padding, margin and line-height. The way how we use these 3 concepts on the container will affect the font size if we are using em. Then be sure with the proprtional scales.

How the browser translate the rem units to pixel values?

When using rem units, the pixel size they translate to depends on the font size of the root element of the page, i.e. the html element (with 16px value). That root font size is multiplied by whatever number you’re using with your rem unit.


How the browser translate the em units to pixel values?

When using em units, the pixel size they translate to depends on the font size of the container element. That container element font size is multiplied by whatever number you’re using with your em unit.


Parent element font sizes can effect em values, but when that happens it’s solely because of inheritance. When we are working with em units, we start to get tricky because of the inheritance rule. The inheritance rule specifies that, every element automatically inherits the font size of its parent. This can only overridden by explicitly setting a font-size with a unit.
The font size of the element on which em units are used determines their size. But that element may have inherited a font size from its parent, which inherited a font size from its parent, and so on. As such it’s possible for any em value to be effected by the font size of any of its parents.


As inheritance, the browser setting also has an effect on element's font size.
By default browsers usually have a font size of 16px, but this can be changed to anywhere from 9px to 72px by the user.


The root html element inherits its font size from the settings in the browser, unless overridden with an explicitly set fixed value. This is where the root element get its default font size.


Summary

At last, hence if you want a resposive site use rem and em on the appropraite part. I recommend you not to use em units in headings.

In addition to this, rem and em for multi column layout widths , instead use % for the fluidly fit. (Even if, the columns can still generally incorporate in its >max-width setting.)

Also don't use rem and em if scaling would unavoidably cause a layout element to break.

The most important thing is that use rem units on media queries.
